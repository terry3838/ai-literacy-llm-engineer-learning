# Exercise Mode - 실습 도전

사용자가 `/mission` 또는 `실습`을 입력했습니다. 아래 절차대로 진행하세요.

## 시작 절차

1. `04_Exercises/` 폴더의 모든 실습을 주차별로 정리하여 제시:

| 주차 | 실습 목록 |
| --- | --- |
| Week01 | AI 윤리 |
| Week02 | 알고리즘, 스택/큐, 트리/그래프, 힙/재귀, 정렬 |
| Week03 | 개발환경, Git, GitHub, Docker |
| Week04 | HTTP, 클라우드, 웹서버, 배포, 네트워크AI |
| Week05 | 프롬프팅, RAG, RAG충돌, 임베딩 |
| Week06 | Tool Calling, Agentic RAG, Context/Safety, Evaluation |
| Week08 | 아키텍처 설계, LangGraph 구현 |
| Week09 | LiteLLM, 상태관리 |

2. "어떤 실습에 도전할까요? (주차 번호 또는 실습 이름)" 질문

## 실습 진행

1. 선택된 실습 노트 (`04_Exercises/*-실습.md`) 읽기
2. `> [!task]` callout의 실습 정보 표시
3. 실습 내용과 핵심 학습 포인트 설명
4. 관련 강의 노트와 개념 노트 안내

## 실습 도전 지원

- 실습 내용을 기반으로 실습 가이드 제공
- 힌트 요청 시 단계별 힌트 제공
- 관련 `exercises/` 폴더의 실습 파일 연결
- 코딩 실습의 경우 코드 리뷰 및 피드백 제공

## 사용 가능 명령어

| 입력 | 동작 |
| --- | --- |
| [실습명] | 해당 실습 상세 보기 |
| 힌트, hint | 현재 실습 힌트 |
| 관련강의 | 실습 관련 강의 노트로 이동 |
| 개념 | 실습 관련 개념 노트 탐색 |
| 목록, list | 전체 실습 목록 다시 표시 |
| 돌아가기, back | 이전 화면으로 |
